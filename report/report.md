# Отчет по домашнему заданию "Алгоритмы поиска подстроки"


## Реализация

Все алгоритмы реализованы на языке C++ с использованием `std::string_view` для избежания копирования строк. Функции возвращают `std::optional<size_t>`, что позволяет явно указать отсутствие результата.
Все реализации в файле ([textScan.cpp](../src/textScan.cpp))
Определения функций в заголовочном файле ([textScan.h](../headers/textScan.h))

1. **Полный перебор (Naive)** — `std::optional<size_t> naiveSearch(std::string_view text, std::string_view pattern)`
2. **КМП (Кнут-Моррис-Пратт)** — `std::optional<size_t> KMPSearch(std::string_view text, std::string_view pattern)`(оптимизация с использованием сдвигов по префиксу шаблона)
3. **Бойер-Мур-Хорспул (BMH)** — `std::optional<size_t> BMHSearch(std::string_view text, std::string_view pattern)` (оптимизация с использованием сдвигов по суффиксу текста)
4. **Бойер-Мур (BM)** — `std::optional<size_t> BMSearch(std::string_view text, std::string_view pattern)` (полная версия с двумя эвристиками)_

## Анализ результатов

### 1. Малые объемы данных (до 100 байт)

На малых объемах данных **все алгоритмы показывают сопоставимое время** (0 мкс с точностью измерения). Это объясняется:
- Низкими накладными расходами на вызовы функций
- Быстрым завершением поиска (успешным или неуспешным)
- Погрешностью измерения микросекундного диапазона

### 2. Большие объемы данных (500000 байт)

На больших текстах разница становится значительной:

| Алгоритм | Время (мкс) | Относительная скорость |
|----------|-------------|------------------------|
| Бойер-Мур | 53 | **13x быстрее Naive** |
| BMH | 57 | **12x быстрее Naive** |
| Полный перебор | 700 | базовый |
| КМП | 852 | медленнее Naive |

### 3. КМП оказался медленнее на случайных данных

КМП демонстрирует **линейную сложность O(n+m)** в теории, но на практике:
- Требует построения префикс-функции (дополнительный проход по шаблону)
- На **случайных данных** плохие символы в BMH дают большие сдвиги
- КМП выигрывает на **повторяющихся паттернах** (например, "aaaaab"), но в случайных данных преимущества не проявляет

### 4. BMH и Бойер-Мур показали почти одинаковые результаты

Для данного теста (случайные данные, шаблон 99 байт):
- Эвристика **плохого символа** уже дает хорошие сдвиги
- Эвристика **хорошего суффикса** добавляет небольшое улучшение (57 мкс -> 53 мкс)
- На **паттернах с повторениями** разница была бы заметнее

## Выводы

1. **Для коротких строк** (до 100 символов) — **любой алгоритм подходит**. Проще использовать полный перебор.

2. **Для длинных случайных текстов** — **Бойер-Мур и BMH дают выигрыш в 10-15 раз**. Это лучший выбор для реальных задач (поиск в логах, текстовых файлах, коде).

3. **Для паттернов с повторениями** (например, "AAAB") — **КМП может быть эффективнее BMH**, но в общем случае уступает Бойер-Муру.

4. **Практическая рекомендация**: использовать **Бойер-Мур-Хорспул** как оптимальный компромисс между:
   - Простотой реализации
   - Высокой производительностью
   - Низкими накладными расходами

5. **Особенности реализации**:
   - Использование `std::string_view` вместо `std::string` для производительности (избегает копирования)
   - `std::optional<size_t>` делает интерфейс безопасным (нет магических чисел)
   - Необходима проверка `pattern.length() > text.length()` во избежание underflow `size_t`
